# サイト構築からMarkdownファイルの読み取り

当初は、ヘッドレスCMSを使ってブログサイトを構築しようとしていました。  
ただ、デザインはテンプレートを使うのではなく、自分でやる方針です。  
そうすると、APIで受け取ったHTMLをデコレーションする必要があります。  
ただ、それはローカルでMarkdownを書いて、変換されたHTMLをデコレーションするのはほぼ同じです。  

コンテンツは自分で書いて管理するので、Markdownで書いて、ローカルに保存するのが一番簡単ということに気づきました。

そこで、一旦サイトをMarkdownで記事を書く方向でサイトを作り直します。
外部サイト[「基礎から始めるGatsbyJS入門」](https://reffect.co.jp/react/gatsby-basic-tutorial-for-beginners)を参照してGatsbyを使ったサイト構築しました。具体的な手順は参照先サイトを見てもらえばわかります。  

## 1. Githubでリポジトリ作成
レポジトリ名を決めて新たにレポジトリを作成します  

```bash:title=bash
$ git clone https://github.com/gituraccount/markdown-blog.git
```

## 2. Gatsbyで空のサイトを作成

ここを参考にして作成しました。  
ここに書いてあるとおりにコマンドを打っていけば出来上がります。
[基礎から始めるGatsbyJS入門](https://reffect.co.jp/react/gatsby-basic-tutorial-for-beginners)

```bash:title=bash
$ gatsby new blog-site https://github.com/gatsbyjs/gatsby-starter-hello-world
```

## 3. サンプル記事を作成
公開用の記事をMarkdownで作成します。

### 3.1 フォルダ構成
ブログ内に画像を貼り付けるため、予めブログ用画像の保存フォルダを作成します。  
srcフォルダ外に`blog/image/`フォルダを作成します。  
各ブログページ用の画像フォルダを別々に作成します。  
例: `blog/image/blog-001-create-blog-site`  

### 3.2 Markdownを加工せずに公開
見た目は悪くても公開できるように、まずはMarkdownのまま公開してみます。  

[GatsbyJSでのGraphQLの使い方](https://reffect.co.jp/react/gatsby-basic-tutorial-for-beginners-2)を参照して進めます。  

そのままやれば表示表示されました。  
ただし、HTMLをきれいに表示するにはもうちょっと工夫が必要のようです。
